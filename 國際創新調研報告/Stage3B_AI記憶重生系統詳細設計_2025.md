## 執行摘要

**AI記憶重生系統**是赤土崎多功能館的核心創新，利用生成式人工智慧、自然語言處理、與圖像生成技術，為每位失智症患者創造**個性化的回憶視頻與互動體驗**。

與傳統回憶療法相比，AI系統具備：
- **規模化**: 可同時服務50-80位長者（傳統1對1療法時間成本高）
- **個性化**: 根據每位長者的人生故事、方言、偏好自動生成內容
- **連續性**: 24/7可用，無需依賴志工與社工時間
- **科學驗證**: 基於2023-2025年國際期刊的VR回憶療法實證

**預期成效**:
- 失智症患者焦慮度↓45%
- 認知功能穩定率↑23%（延遲衰退18-24個月）
- 家屬遠距陪伴滿意度↑87%

## 一、國際證據基礎

### 1.1 回憶療法的科學有效性

根據Wiley Scholar Gateway 2023-2025年度系統性回顧：

**Ng et al. (2024) — 虛擬實境回憶療法的系統性回顧**
發表於 *Journal of Clinical Nursing*, DOI: 10.1111/jocn.17375

- **研究規模**: 22項實驗性研究薈萃分析
- **樣本**: 1,200+名60歲以上長者
- **主要發現**:
  - VR回憶療法對**焦慮症改善**: 平均改善度 58.3% (p<0.001)
  - 對**抑鬱症改善**: 平均改善度 42.7% (p<0.01)
  - 對**認知結果的改善**: 記憶保留 48-65%

**核心結論**: 高沉浸式VR環境、可個性化、具自主控制權的回憶體驗，效果**顯著優於靜態照片或音樂回憶**。

**Bayram (2023) — 失智症患者的回憶療法**
發表於 *Psychogeriatrics*, DOI: 10.1111/psyg.13052

- **實驗組**: 27名失智症患者，每週2次、各45分鐘回憶療法
- **控制組**: 27名無干預
- **3個月結果**:
  - MMSE認知評分: 實驗組 ↑3.2分，控制組 ↓0.8分
  - 焦慮度(CMAI量表): 實驗組 ↓45%，控制組 ↓8%
  - 生活品質評分: 實驗組 ↑28%, 控制組 ↓5%

**應用洞見**: **結構化的回憶療法**（非隨意回憶）對失智症患者的認知保護效果最佳。

### 1.2 生成式AI在長照應用的現況

**GoodTimes AI Photo Album (2024)**
發表於 *JMIR Aging*, 2024年1月
DOI: 10.2196/49415

**技術概述**:
- 使用AI進行照片識別（Google Vision API）
- 自動生成故事敘述（GPT-3.5與微調模型）
- 與長者進行自然對話（對話管理系統）

**臨床結果** (20名認知完整長者, 8週試用):
- **參與度**: 平均每週使用2.3小時（相比預期1小時高出130%）
- **滿意度**: 18/20受試者表示「有幫助」或「非常有幫助」
- **心理效益**: 報告的孤立感↓32%, 幸福感↑38%

**技術成熟度**: **TRL 7級**（原型驗證完成，可擴展）

**其他國際案例**:

1. **Synthetic Memories Project (英國, 2022-2025)**
   - 使用DALL-E 3和Midjourney生成失去的記憶相片
   - 與英國NHS進行臨床試驗
   - 初步結果：患有阿茲海默症的受試者在回顧合成回憶時有「恢復遺失時刻」的感受

2. **Valinor AI (美國, 2025)**
   - LLM對話機器人捕捉長者的生活故事
   - 提供個性化的記憶提示與對話
   - 用戶數據：60%參與度提升，滿意度評分↑25%

3. **Music-Generative AI for Reminiscence (日本, 2024)**
   - 發表於 *Brain and Behavior*, DOI: 10.1002/brb3.70007
   - 使用MusicTGA-HR (Music Trinity Generative Algorithm)
   - 118名社區長者參與6個月
   - **結果**: 詞彙流暢度↑12% (p=.003)，Trail-Making Test反應時間↓18% (p=.039)

## 二、赤土崎AI記憶重生系統的技術架構

### 2.1 系統整體流程

```
長者入住赤土崎館
    ↓
[1] 個人檔案建立
    ├─ 生活史訪談 (4-6小時)
    │  └─ 記錄: 出生地、童年、工作、家庭、愛好、方言、重要時刻
    ├─ 相片與視頻上傳
    │  └─ 數位化: 舊相簿、家族影片、紀念照片
    └─ 情感偏好評估
       └─ 問卷: 喜歡什麼音樂、食物、人物、場景

    ↓
[2] AI模型微調
    ├─ LLM微調 (LoRA)
    │  └─ 在GPT-4或Claude基礎上，微調長者的「說話方式」
    │     例: 台灣台語、客家話、港澳粵語等方言
    ├─ 情感分析模型
    │  └─ 訓練模型識別長者的焦慮、喜悅、困惑等情緒
    └─ 向量資料庫
       └─ 儲存長者的人生事件、重要人物、地點向量

    ↓
[3] 回憶視頻生成
    ├─ 內容規劃 (AI自動)
    │  ├─ 主題選擇: 「你童年在台南的故事」
    │  ├─ 敘述腳本: 使用微調LLM生成「如果我是你，我會這樣說...」
    │  └─ 視覺元素: 使用DALL-E 3生成「1950年台南街景」圖像
    │
    ├─ 視頻合成
    │  ├─ 文字轉語音 (TTS): 使用與長者方言相符的合成嗓音
    │  ├─ 圖像搜尋: 從Google Images搜尋歷史照片
    │  ├─ 圖像融合: 使用Stable Diffusion或DALL-E重構回憶場景
    │  └─ 動畫與過渡: 使用FFmpeg與MoMa技術生成4K品質視頻
    │
    └─ 品質檢驗
       ├─ 家屬審核: 確保內容準確、恰當
       └─ AI偏差檢測: 確保沒有製造虛假記憶

    ↓
[4] 互動投影體驗
    ├─ 播放: 在2F失智專區、1F共用活動室投影
    ├─ 互動: 長者可用手勢或聲音控制
    │  ├─ 「下一個故事」(手勢)
    │  ├─ 「告訴我更多」(語音)
    │  └─ 「我想起來了...」(語音捕捉，記錄長者反應)
    └─ 家屬遠距參與: APP即時推播，父母可在公司同步觀看

    ↓
[5] 療效追蹤
    ├─ 生理指標: HR, SpO2, 皮膚溫度 (穿戴設備)
    ├─ 行為指標: 笑容辨識、身體活動度 (攝像機+AI)
    ├─ 認知評估: 每周進行簡短MMSE測試
    └─ 家屬反饋: 每周問卷評估長者的情緒、睡眠、食慾
```

### 2.2 核心技術選擇

#### A. 大語言模型(LLM)選擇

| LLM模型 | 優勢 | 劣勢 | 選擇 |
|--------|------|------|------|
| **GPT-4o** | ✓最強推理能力, 支援LoRA微調 | ✗最高成本 ($15/1M output tokens) | ⭐ 首選 |
| **Claude 3.5 Sonnet** | ✓記憶力強, 台灣可用 | ✗暫不支援LoRA | ✓ 備選 |
| **Llama 2 (開源)** | ✓無成本, 可在地部署 | ✗推理能力弱於GPT-4 | ✓ 輔助 |

**赤土崎選擇**: GPT-4o (透過OpenAI API) + 本地Llama 2備援

#### B. 圖像生成模型

| 模型 | 速度 | 品質 | 成本 | 用途 |
|------|------|------|------|------|
| **DALL-E 3** | 中 (30秒) | 高 (4K) | ✓$0.04-0.08/張 | 主要生成 |
| **Midjourney** | 慢 (1-2分鐘) | 最高 | $0.30-0.40/張 | 精品場景 |
| **Stable Diffusion** | 最快 (5秒) | 中 | 免費 (開源) | 快速原型 |

**赤土崎選擇**: DALL-E 3 (成本與品質平衡) + Stable Diffusion (備援)

#### C. 文字轉語音 (TTS) 與方言支援

**需求**: 支援台灣台語、客家話、粵語、標準漢語

| 服務 | 方言支援 | 自然度 | 成本 |
|-----|---------|--------|------|
| **Azure Speech Services** | ✓台語、客語、粵語 | 中 | $1-5/1K字 |
| **Google Cloud TTS** | △基礎台語 | 高 | $4/1M字 |
| **本地Tacotron2** | ✓所有方言 | 中 | 免費 |

**赤土崎選擇**: Azure Speech Services (方言完整度最佳) + 本地Tacotron2備援

## 四、倫理與隱私框架

### 4.1 AI生成內容的倫理準則

#### A. 「誠實回憶」原則

**定義**: 不製造虛假或扭曲的回憶，所有生成內容必須基於長者自述或確實的歷史

**實施方式**:
```
內容驗證流程:

生成的回憶內容
    ↓
[檢查1] 與長者自述的匹配度 ≥95%
    ├─ 是否與訪談記錄相符?
    ├─ 是否有任何創意虛構?
    └─ 方言、口氣是否正確?
    ↓
[檢查2] 家屬驗證簽核
    ├─ 家屬確認內容的真實性
    ├─ 檢查是否包含敏感或傷痛內容
    └─ 是否需要編輯或刪除部分
    ↓
[檢查3] 心理健康評估
    ├─ 此內容可能激發的情緒是正面還是負面?
    ├─ 是否涉及創傷記憶?
    └─ 長者心理素質是否適合呈現?
    ↓
核准 → 發布 / 修改 / 駁回
```

#### B. 知情同意

**要求**:
1. 長者本人簽署「AI回憶療法同意書」(簡體中文、繁體中文、粵語、台語版本)
2. 家屬也需簽署，了解:
   - 生成內容的技術原理
   - 隱私數據的使用與儲存方式
   - 可隨時要求刪除所有資料
3. 每年重新同意，確保長者意願的持續性

**不能同意的情況**:
- 長者無行為能力或無法理解同意書
- 家屬有強烈反對
- 倫理委員會認為該長者不適合此療法

#### C. 隱私與資料安全

**資料分類**:
```
[頂級機密] 長者的人生故事音頻、舊相簿、家族資訊
├─ 儲存: 本地加密伺服器 (不上雲端)
├─ 訪問: 僅限該長者的醫療團隊 + 授權家屬
├─ 日誌: 所有訪問都被記錄(誰在何時訪問什麼資料)
└─ 銷毀: 長者要求或合約終止後90天內銷毀

[高度機密] 生成的回憶視頻
├─ 儲存: 加密雲端備份 (AWS S3 + 加密)
├─ 訪問: 限該長者、家屬、直接照護人員
└─ 有效期: 長者在世期間 + 1年, 之後自動刪除

[一般機密] 療法效果數據 (去識別化)
├─ 用途: 臨床研究與學術發表
├─ 訪問: 研究人員(需倫理委員會審批)
└─ 發表: 已去識別化，無法追蹤到個人
```

**技術措施**:
- 256-bit AES加密 (儲存與傳輸)
- 多因素認證 (MFA) for 所有系統訪問
- 年度資安滲透測試
- GDPR與台灣個資法合規

## 六、國際案例對標

### 6.1 GoodTimes (美國, 2024)

**技術棧**: GPT-3.5 + Google Vision API + Next.js Web App

**成果** (20名認知完整長者):
- 每週使用2.3小時 (預期1小時，提升130%)
- 滿意度9/10
- 孤立感↓32%

**赤土崎改進點**:
- ✓ 針對失智症患者(非認知完整者)
