## 📸 如何截圖（用於簡報）

### 方法 1：Windows 內建工具（最快）
1. 按 `Win + Shift + S`
2. 選取要截圖的區域
3. 圖片會自動複製到剪貼簿
4. 貼到PowerPoint即可（Ctrl+V）

### 方法 2：瀏覽器全螢幕截圖（高品質）
1. 按 `F12` 開啟開發者工具
2. 按 `Ctrl + Shift + P` 開啟命令面板
3. 輸入 `screenshot`
4. 選擇 `Capture full size screenshot`
5. 圖片會自動下載

### 建議解析度
- **簡報用**：1920x1080
- **海報用**：3840x2160（4K）
- **網頁用**：1280x720

## ⚠️ 常見問題

**Q: `npm install` 失敗？**
A: 確認已安裝Node.js（v16+）：`node -v`

**Q: 圖表顯示不出來？**
A: 清除快取後重新整理（Ctrl+Shift+R）

**Q: 想要修改顏色？**
A: 編輯 `PainPointDashboard.jsx`，搜尋 `color: '#ff6384'` 並修改

**Q: 可以離線展示嗎？**
A: 執行 `npm run build`，然後用 `npm run preview`

**Q: 圖表太小？**
A: 調整瀏覽器縮放（Ctrl + 滾輪）或修改組件的 `width`/`height`

## 🎤 現場展示 vs 靜態截圖

### 現場展示（推薦給自信的人）
**優點**:
- 互動感強，可即時調整
- Tooltip很炫，增加專業感
- 評審可能會「哇」一聲

**缺點**:
- 網路可能斷線
- 筆電可能卡頓
- 需要熟練操作

**建議**: 只在Q&A環節展示，簡報主體用截圖

### 靜態截圖（推薦給保守的人）
**優點**:
- 穩定，不會出錯
- 可以精心排版
- 簡報檔案小

**缺點**:
- 缺乏互動感
- 無法即時調整

**建議**: 主要使用截圖，準備一個瀏覽器分頁當備案
