# 資料清理與增強任務

**更新日期**: 2025-10-22
**目標**: 為114年新竹政策黑客松提案準備高品質資料集

---

## 📋 任務總覽

### 階段一：基礎清理（Data Cleaning）

#### 1. 去重（Deduplication）
- **方法**: 使用 cosine similarity 計算文章相似度
- **閾值**: 相似度 > 0.85 視為重複
- **處理**: 保留較完整或較新的版本

#### 2. 去噪（Noise Removal）
- 移除廣告內容
- 移除無關社群討論（如網站導航、留言板功能按鈕）
- 過濾非竹科家庭相關內容

#### 3. 文本標準化（Text Normalization）
- 統一日期格式：YYYY-MM-DD
- 移除 HTML 標籤殘留
- 繁簡轉換：統一為繁體中文
- 統一單位表示（萬元、億元）

---

### 階段二：情緒與強度標註（Emotion & Intensity Annotation）

#### 4. 情緒分析（Sentiment Analysis）
**新增欄位**: `emotion_score`
- **類型**: Float (-1.0 到 1.0)
- **定義**:
  - -1.0 到 -0.3: 負面（如憤怒、焦慮、絕望）
  - -0.3 到 0.3: 中性（如陳述事實、資訊分享）
  - 0.3 到 1.0: 正面（如希望、感激、滿意）

**使用模型**:
- 中文情緒分析模型（如 BERT-based sentiment model）
- 針對竹科家庭特定詞彙微調（如「人生貧戶」、「複製人家庭」）

#### 5. 痛點強度識別（Pain Point Intensity）
**新增欄位**: `pain_point_intensity`
- **類型**: String ["高", "中", "低"]
- **判斷標準**:
  - **高**: 涉及安全風險、心理健康問題、關係破裂（如托嬰中心溺斃、社群排擠、離婚）
  - **中**: 長期壓力、經濟負擔、時間壓力（如通勤2小時、托育費用3萬/月）
  - **低**: 生活不便、選擇受限（如餐廳選擇少、假日無聊）

---

### 階段三：實體識別與結構化（Entity Extraction & Structuring）

#### 6. 地理實體識別（Location Entities）
**新增欄位**: `entities.locations`
- **類型**: Array of Strings
- **提取對象**:
  - 新竹市行政區：東區、北區、香山區
  - 竹科相關：竹科、科學園區、園區、新竹科學園區
  - 跨縣市通勤：竹北、竹東、頭份、桃園、八德、青埔
  - 地標：巨城、關東市場、護城河、清華大學、交通大學
  - 道路：東科路、光復路、金橋交通燈

#### 7. 機構實體識別（Organization Entities）
**新增欄位**: `entities.organizations`
- **類型**: Array of Strings
- **提取對象**:
  - 科技公司：台積電、聯發科、聯電、友達、華碩
  - 政府單位：新竹市政府、社會處、衛生局、長期照顧管理中心
  - 教育機構：竹科實中、清華附小、華德福幼兒園
  - 社福機構：老五老基金會、伊甸基金會
  - 托育機構：公共托嬰中心、托育資源中心

#### 8. 人物角色識別（Person Role Entities）
**新增欄位**: `entities.person_roles`
- **類型**: Array of Strings
- **角色分類**:
  - 職業身份：竹科媽媽、工程師、RD（研發工程師）、產線人員
  - 家庭角色：雙薪家庭、單親家長、隔代教養、全職媽媽
  - 年齡群體：新生兒、幼兒、學齡兒童、青少年、長輩、失智長者
  - 社會群體：北漂人、台北人、竹科人、高學歷家長

---

### 階段四：資料品質與可信度評估（Data Quality & Credibility）

#### 9. 資料來源可信度標註（Credibility Assessment）
**新增欄位**: `credibility`
- **類型**: String ["高", "中", "低"]
- **評估標準**:
  - **高**: 政府官方公告、主流新聞媒體、學術研究、統計數據
  - **中**: 個人部落格、社群媒體（有身份認證）、地方新聞
  - **低**: 匿名社群討論、未查證的轉載、情緒性發文

#### 10. 資料更新與驗證（Data Update & Validation）
**新增欄位**: `update_date`
- **類型**: String (YYYY-MM-DD)
- **說明**: 資料最後驗證日期
- **更新機制**:
  - 定期驗證2025年最新數據
  - 追蹤政策進度更新（如赤土崎綜合福利館標案進度）
  - 標註已過時或失效的資訊

---

### 階段五：政策對接與主題增強（Policy Mapping & Topic Enhancement）

#### 11. 政策對接點標註（Related Policy Mapping）
**新增欄位**: `related_policy`
- **類型**: Array of Strings
- **政策分類**:
  - **長照2.0**: 涉及日照中心、長照服務、失智照護
  - **托育補助**: 涉及公共托嬰中心、托育費用、育兒津貼
  - **財劃法**: 涉及市府預算、統籌分配稅款、社福建設
  - **社福中心**: 涉及社會福利服務、一區一中心政策
  - **竹科效應**: 涉及人口成長、高薪族群、生育潮
  - **一學區一日照**: 涉及日照中心布建、ABC長照網絡

#### 12. 主題分類優化（Topic Classification Enhancement）
**現有主題**（保留）:
- 托育/教育壓力
- 長照/失智照護
- 工作-家庭平衡
- 社群焦慮/教養競爭
- 財務壓力

**新增主題**:
- **時間貧窮**: 通勤時間、接送時間、家庭時間不足
- **跨縣市通勤**: 頭份/竹北/桃園通勤竹科的困境
- **社福資源分散**: 服務存在但地點分散，接送不便
- **心理健康**: 憂鬱、焦慮、慢性處方箋、諮商需求
- **社群排擠**: 教育理念衝突、社交孤立、複製人家庭現象
- **托育安全**: 托嬰中心溺斃、安全監管、師生比問題
- **生活品質**: 生活機能、娛樂選擇、居住環境

---

### 階段六：進階標註（Advanced Annotation）

#### 13. 痛點類型細分（Pain Point Categorization）
**新增欄位**: `pain_point_categories`
- **類型**: Array of Strings
- **分類**:
  - **時間類**: 通勤耗時、接送困難、等待時間、時間碎片化
  - **金錢類**: 托育費用、房價壓力、雙車家庭、補習費
  - **心理類**: 焦慮、壓力、社交孤立、教養競爭、身份認同
  - **關係類**: 夫妻衝突、親子疏離、社群排擠、家庭失和
  - **健康類**: 慢性病、睡眠不足、心理疾病、安全風險
  - **資源類**: 托育不足、長照分散、交通不便、資訊不對稱

#### 14. 關鍵引文提取（Key Quote Extraction）
**新增欄位**: `key_quotes`
- **類型**: Array of Strings
- **提取標準**:
  - 具代表性的第一人稱陳述
  - 具感染力的情緒表達
  - 具體的數據或案例
  - 可用於黑客松簡報的金句

**範例**:
```json
"key_quotes": [
  "年薪300萬在新竹其實是甲級貧戶",
  "我孩子為什麼媽媽你今天不用在公司？",
  "小學考90分以下就是全班倒數啦",
  "托嬰中心19:55關門，老師用力拉下鐵門"
]
```

#### 15. 解決方案對應（Solution Mapping）
**新增欄位**: `solution_mapping`
- **類型**: Object
- **結構**:
```json
"solution_mapping": {
  "direct_solution": "赤土崎綜合福利館可直接解決的痛點",
  "indirect_benefit": "間接改善的問題",
  "synergy_effect": "與其他服務整合的綜效"
}
```
